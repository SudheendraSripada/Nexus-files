03 — Replit Python Auth & Forwarding Engine
Python Logic

This is the "Face" of the app. It handles the OAuth, saves the user, and securely passes data to Make.com.

Act as an expert Python backend developer. Build a Flask backend for my app "NEXUS". Constraint: Ensure the code is procedural and easy to read. Use nested if-else statements for routing logic where applicable. ARCHITECTURE GOAL: This app sits between the User and Make.com. It handles the OAuth login for LinkedIn/Google, stores the token, and provides a webhook receiver that forwards verified payloads to Make.com. ENDPOINTS REQUIRED: 1. `GET /auth/`: - Initiates standard OAuth 2.0 flow for 'linkedin' or 'gmail'. Redirects to the respective provider. 2. `GET /callback/`: - Catches the code, exchanges it for an access_token. - Saves the access_token securely into a Supabase database table called `user_tokens`. - Redirects the user to `/success`. 3. `POST /nexus-receiver`: - This endpoint receives incoming comment data (e.g., from an Apify scraper). - Extracts the `user_id`, `platform`, `comment_text`, and `profile_url`. - Logic: If `comment_text` contains "DEPLOY": - Fetch the user's stored access_token from Supabase. - Package the token, platform, and profile_url into a JSON payload. - Use the `requests` library to POST this JSON to my Make.com Custom Webhook URL. Else: - Ignore the comment and return a 200 OK.
